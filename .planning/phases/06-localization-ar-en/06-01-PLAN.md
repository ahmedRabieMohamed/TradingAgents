<task type="auto">
  <name>Task 1: Create localization helpers for analytics results</name>
  <files>tradingagents/api/services/localization.py</files>
  <action>
    - Add a new module `localization.py` under `tradingagents/api/services/`.
    - Define a `LanguageCode` Enum (`en`, `ar`) for query validation.
    - Create translation maps for deterministic strings:
      - Decision labels: BUY/SELL/HOLD.
      - Trend/momentum/volatility values: uptrend/downtrend/mixed, overbought/oversold/positive/negative/neutral, high/moderate/low.
      - Support/resistance labels and notes (support/resistance/pivot; 20d rolling low/high, pivot-derived, pivot point).
      - Liquidity anomaly labels (high_volume_spike/low_volume_drop) and severities (high/moderate).
      - Report headings and bullet labels used in `final_report` (e.g., "## Technical Indicators", "## Decision", "- Label:", etc.).
    - Implement helper functions that localize structured fields without mutating stored payloads:
      - `localize_summary(summary, lang)` to translate `trend`, `momentum`, `volatility`, and rebuild `summary.summary` when possible by parsing the ATR/Close ratio from the English text (fallback to original if parsing fails).
      - `localize_risk_notes(notes, lang)` to translate deterministic risk note sentences, parsing dynamic percentages where present.
      - `localize_decision(decision, lang)` to translate `label` and rewrite `rationale` using the known template (parse bullish/bearish counts and RSI if present; fallback to original if parsing fails).
      - `localize_support_resistance(levels, lang)` and `localize_liquidity_anomalies(anomalies, lang)` to translate labels/notes/descriptions, handling the known English templates with regex.
      - `localize_final_report(text, lang)` to translate markdown headings/labels while leaving agent-generated narrative paragraphs intact.
    - Provide a single entrypoint `localize_analytics_result(result: AnalyticsReportResult, lang: LanguageCode)` that returns a localized copy (use `model_copy(deep=True)` and reassign localized fields). Return the original result unchanged when `lang` is `en`.
  </action>
  <verify>python -m compileall tradingagents/api/services/localization.py</verify>
  <done>Localization helpers exist with deterministic string mappings and a `localize_analytics_result` entrypoint.</done>
